Centralized(SVN) vs De-Centralized (GIT)


1- workflow--

SVN-- check out -- edit -- check in to central server

GIT -- clone -- edit -- commit(to local)--push to remote(Github/GitLab/BitBucket)

2- 

SVN- centralized VCS- Full source code at server only.

GIT- De-centralized VCS-- Full source code copy at each developer system.

3-

SVN- Internet required 

GIT - Offline work and later sync with remote.

4-

SVN - single point of failure - server 

GIT - Multiple copies available so safe.

Consumer access the queue

consumer process the queue message and deletes.

if messages are failed multiple times they stores at DLQ.
